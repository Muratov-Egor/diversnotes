# Теги

Единый список тегов хранится в **content/tags.json**. В статьях и в Marine Life используются только теги из этого списка.

---

## 1. Формат content/tags.json

Массив объектов с полями:

- **slug** — идентификатор для URL (латиница, без пробелов). Пример: `/tags/diving`
- **ru** — подпись на русском, которую указывают в frontmatter постов

Пример:

```json
[
  { "slug": "diving", "ru": "дайвинг" },
  { "slug": "first-experience", "ru": "первый опыт" }
]
```

Позже, при включении EN, можно добавить поле **en** для английской подписи.

---

## 2. Как использовать в постах

В frontmatter блога и Marine Life в поле **tags** указываются подписи **на русском** (значения **ru** из tags.json):

```yaml
tags:
  - дайвинг
  - первый опыт
  - пхукет
```

Не использовать: slug в frontmatter, теги, которых нет в content/tags.json, или с опечатками.

---

## 3. Добавление нового тега

1. Открыть **content/tags.json**.
2. Добавить объект: `{ "slug": "url-slug", "ru": "Подпись по-русски" }`.
3. Сохранить. После этого тег можно использовать в постах.

slug должен быть уникальным и в латинице (для URL).

---

## 4. Обновление старых статей (миграция)

При переносе или правке старых статей:

1. Открыть **content/tags.json** и посмотреть текущий список разрешённых тегов.
2. В каждой статье в frontmatter в **tags** оставить только теги, которые есть в tags.json (поле **ru**).
3. Если нужного тега нет — сначала добавить его в **content/tags.json**, затем использовать в статье.
4. Привести к одному написанию: например, «дайвинг», а не «Дайвинг» или «дайвинг » (без лишних пробелов).
5. После изменений проверить сборку (при наличии валидации — что все теги постов проходят проверку).

Код: **lib/content/tags.ts** — `getAllTags()`, `getTagBySlug(slug)`, `isAllowedTag(ruLabel)` для страниц и валидации.
