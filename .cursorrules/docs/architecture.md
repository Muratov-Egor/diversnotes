# Архитектура проекта DiversNotes

## 1. Общая стратегия

Проект переписывается с нуля (greenfield).
Цель — создать минималистичный, быстрый, SEO-структурированный авторский журнал с долгосрочной поддержкой.

Основной язык: RU (без префикса).
EN временно отключён, но архитектура должна поддерживать его возврат.

---

## 2. Технологический стек

- Next.js 14+ (App Router)
- TypeScript (strict mode)
- MDX для контента
- Tailwind CSS
- Static Site Generation (по умолчанию)
- next/image для медиа

Контент хранится в Git. CMS не используется.

---

## 3. Структура проекта

scripts/
docs/

---

## 4. Принципы разработки

- Server Components по умолчанию.
- Клиентские компоненты только при необходимости (поиск, переключатель темы).
- Валидация контента на этапе build.
- Никакой избыточной клиентской логики.
- Минимум внешних зависимостей.
- Код должен быть понятен и поддерживаем одним человеком.

---

## 5. Производительность

Приоритет: мобильная версия.

Обязательные правила:

- next/image
- AVIF/WebP
- lazy loading
- корректный sizes
- отсутствие тяжёлых клиентских библиотек

Mobile Lighthouse должен быть высоким.
